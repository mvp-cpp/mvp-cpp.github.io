Section 1

Introduction

MVP is a memory-efficient, visualization-enhanced and parallel-accelerated command-line tool for genome-wide association study. This tutorial follows the style of a practical software manual: prepare input files, select an analysis mode, run MVP, then inspect outputs.

What is MVP

MVP is implemented in C++17 for scalable GWAS workflows. It is designed as a command-line program: each run starts from one major function flag, reads well-defined input files, and writes analysis tables or plots under the output prefix. The same phenotype and genotype files can be reused across GLM, MLM, PCA, GRM construction and downstream population-aware workflows.

  • --glm: General Linear Model.
  • --mlm: Mixed Linear Model.
  • --pca: Principal component analysis.
  • --make-grm: Genomic relationship matrix construction.
  • --cluster or --auto-cluster: Unsupervised adaptive population clustering.
  • --ucalm: Unsupervised adaptive Clustering-based Association and Layered Meta integration.
  • --meta: Meta-analysis for multiple GWAS summary statistics files.
  • --plot and --plot-only: Visualization after analysis or from existing results.

Typical workflow. Prepare phenotype and genotype files, run PCA or GRM if needed, select GLM or MLM for association analysis, then use plotting or meta-analysis commands to summarize the results.

Main Features

  • MVP accepts PLINK 1 binary genotype prefixes with --bfile and PLINK 2 PGEN prefixes with --pfile.
  • Multi-trait phenotype analysis with comma-separated phenotype columns.
  • CPU execution by default, optional CUDA acceleration for supported builds.
  • Automatic plot generation for association and PCA outputs when plotting is enabled.
  • Optional compile-time modules for clustering, meta-analysis and plotting.
FeatureWhat it is used forMain options
Association scanSingle-trait or multi-trait GWAS with additive or dominant effects.--glm, --mlm, --add, --dom
Population structurePCA correction, clustering, subgroup GWAS and meta-analysis.--npc, --cluster, --pop-class

Command Structure

MVP uses one main function flag followed by input, model, runtime and output options. In most cases, use only one function flag in a command. For example, do not run --pca and --make-grm in the same command. 

mvp [Function] [Option]

For example:

mvp --glm --bfile data/geno --pheno data/pheno.txt --out result/glm
Option groupExamplesNotes
Function--glm, --mlm, --pca, --metaSelects the major analysis mode.
Input--pheno, --bfile, --pfile, --grmPoints to phenotype, genotype, GRM or summary files.
Model--pheno-pos, --dcovar, --qcovar, --mafDefines trait columns, covariates and filtering.
Runtime--threads, --device, --stepControls CPU/GPU resources and block size.
Output--out, --plot, --benchmark-outControls output prefix, figures and benchmark records.

Check Version and Help

mvp --version
mvp --help

Use --help before starting a new analysis to confirm which optional modules are available in your build. If a flag shown in this tutorial is missing from the help message, rebuild MVP with the corresponding module enabled or use a release binary that includes that feature.

Reproducibility tip. Save the output of mvp --version, the full command line and the input file prefixes together with your result files.

Section 2

Input File Formats

MVP follows HIBLUP-compatible data organization. Keep individual IDs consistent across phenotype, genotype, PC, GRM and population files.

Phenotype File

The phenotype file is a text table with a header. The first column is the individual ID. Missing values should be encoded as NA. Phenotype column positions are 1-indexed. 

id trait1 trait2 sex year height
ID001 12.4 8.1 F 2025 168
ID002 NA 7.6 M 2025 175
ID003 10.9 8.4 F 2026 162

Use --pheno-pos to select the analyzed traits. The default is column 2. 

--pheno demo.phe --pheno-pos 2
--pheno demo.phe --pheno-pos 2,3
ColumnMeaningHow MVP uses it
Column 1Individual IDUsed to match genotype, PC, population and GRM IDs.
Trait columnsNumeric phenotypesSelected by --pheno-pos. Multiple columns can be comma-separated.
Discrete covariatesGroups such as sex, year or batchSelected by --dcovar.
Quantitative covariatesContinuous variables such as height or ageSelected by --qcovar.

Column numbering. MVP follows user-facing 1-based column positions. Because the first column is reserved for IDs, phenotype positions must be larger than 1.

Genotype Files: PLINK BED and PLINK2 PGEN

MVP supports both PLINK 1 binary files and PLINK 2 PGEN files. Provide the file prefix, not the file name with extension. The sample IDs in genotype files should match the IDs in the phenotype file after any keep/remove filters are applied. 

# PLINK 1 binary prefix
demo.bed
demo.bim
demo.fam
--bfile demo

# PLINK 2 PGEN prefix
demo.pgen
demo.pvar
demo.psam
--pfile demo
Input typeRequired filesMVP optionCommon mistake
PLINK 1 binary.bed, .bim, .fam--bfile demoUsing --bfile demo.bed instead of the prefix.
PLINK 2 PGEN.pgen, .pvar, .psam--pfile demoSupplying --bfile and --pfile together.

Covariates in Phenotype File

MVP reads covariates from the phenotype file by column position. Use --dcovar for discrete covariates and --qcovar for quantitative covariates. 

# column 4 = sex, column 5 = year, column 6 = height
--dcovar 4,5
--qcovar 6
  • Discrete covariates are converted to model terms suitable for categorical variables.
  • Quantitative covariates are treated as numeric columns and should not contain text labels.
  • Do not include the phenotype column itself in --dcovar or --qcovar.
  • When a covariate has many missing values, remove or impute those records before running MVP.

Principal Component File

A PC file can be used as standalone clustering input with --cluster. It should include a header, and the first column should contain individual IDs. For GLM or MLM, PCs are usually generated internally by specifying --npc. 

id PC1 PC2 PC3
ID001 -0.021 0.114 0.008
ID002 0.034 -0.090 0.012
ID003 -0.006 0.041 -0.017

To generate PCs from genotype data, use --pca. To use an existing PC file as standalone clustering input, use --cluster --pc. 

--pca --bfile demo --npc 5
--cluster --pc result/demo_pca.pc --npc 3

Important. --pc is for standalone clustering input. For GLM or MLM correction, use --npc so MVP can add principal components to the model.

Population Classification File

Use --pop-class to provide known population labels. The file contains two columns: individual ID and population label. 

ID001 POP_A
ID002 POP_B
ID003 POP_A

When this file is supplied with --glm or --mlm, MVP can run population-specific association scans and then combine subgroup evidence through the meta-analysis workflow if the meta module is available.

  • Each individual should appear at most once.
  • Population labels should be short and stable because they may be used in output column names.
  • --pop-class cannot be used together with standalone --cluster.
  • Do not provide a pre-computed --grm in the same command when using population classification.

Relationship Matrix File

Use --grm with the prefix of pre-computed relationship matrix files. MVP expects paired binary and ID files. A GRM can be reused by MLM and PCA, which is useful when several traits share the same genotype data. 

demo_grm.bin
demo_grm.id

--grm demo_grm

ID consistency. The GRM ID file must match the individuals used in the phenotype and genotype data after sample filtering.

GWAS Summary Statistics File

Use --sumstat for standalone meta-analysis. At least two summary statistics files are required. Each file should include the following header fields: 

SNP chr pos a1 a2 freq b se p n
rs001 1 10583 A G 0.42 0.031 0.010 0.0021 1200
rs002 1 12940 C T 0.18 -0.024 0.012 0.0450 1200
FieldDescriptionRequirement
SNPMarker ID.Should be consistent across cohorts whenever possible.
chr, posChromosome and base-pair position.Use the same genome build across all files.
a1, a2Effect and non-effect allele coding.Allele orientation must be checked before meta-analysis.
freqAllele frequency for a1.Used for MAF and population-structure-aware methods.
b, se, pEffect, standard error and p-value.Required for method-specific meta-analysis statistics.
nSample size.Should be numeric and cohort-specific.

Meta-analysis input. --sumstat requires at least two files, separated by commas, and duplicate file paths are rejected.

Genome Window File

Window-based methods can use fixed windows or a user-defined window file. The user-defined file should include chr, start and end columns. 

chr start end
1 1 1000000
1 1000001 2000000
2 1 1000000

Window options are mutually exclusive:

# Fixed physical window size
mvp --meta --sumstat c1.sumstat,c2.sumstat --window-bp 1000000 --out result/meta_bp

# Fixed number of SNPs per window
mvp --meta --sumstat c1.sumstat,c2.sumstat --window-num 500 --out result/meta_num

# User-defined window file
mvp --meta --sumstat c1.sumstat,c2.sumstat --window-file windows.txt --out result/meta_windows
  • If no window option is supplied, MVP uses a default fixed base-pair window.
  • Do not use --window-bp and --window-num together.
  • Do not provide fixed window settings when --window-file is used.

Sample and Marker Filter Files

Use --keep and --remove for individual filters. Use --extract and --exclude for marker filters. Do not specify both include and exclude files for the same target. 

# Keep only selected individuals
--keep keep.id

# Remove selected individuals
--remove remove.id

# Analyze selected markers
--extract marker_keep.txt
TargetInclude optionExclude optionExpected content
Individuals--keep--removeOne individual ID per line.
Markers--extract--excludeOne marker ID per line.

Quality Control

Before running large analyses, check ID consistency, missing phenotype values, genotype file prefixes, MAF thresholds, sample filters and marker filters. Use --maf to set the minor allele frequency threshold.

  • Confirm that phenotype IDs and genotype IDs use the same naming convention.
  • Check that the first phenotype column is the ID column and all selected trait columns are numeric or NA.
  • Remove samples or markers with known quality issues before running production analyses.
  • Use a clear --out prefix for each test run so intermediate files are not overwritten.
  • For meta-analysis, check genome build, marker ID, allele coding and sample-size columns before combining cohorts.
Section 3

Running MVP

This section lists common MVP workflows. Use one analysis mode per command unless the workflow explicitly supports automatic clustering inside GLM or MLM.

Quick Start

Start with a small and explicit command before launching a full analysis. The examples below assume that genotype files use the prefix data/geno, the phenotype file is data/pheno.txt, and output files will be written under result/. 

# GLM GWAS
mvp --glm --bfile data/geno --pheno data/pheno.txt --out result/glm

# MLM GWAS
mvp --mlm --bfile data/geno --pheno data/pheno.txt --out result/mlm

# PCA
mvp --pca --bfile data/geno --npc 5 --out result/pca

# GRM construction
mvp --make-grm --bfile data/geno --out result/grm
  • Use --glm first to verify basic phenotype, genotype and covariate formatting.
  • Use --mlm when relatedness or population structure needs stronger control.
  • Use --pca or --make-grm separately when you want to inspect intermediate files.
  • Use a fresh --out prefix for each run to avoid overwriting previous output.

Output Prefix and Threads

Use --out to set the output prefix and --threads to set the number of CPU threads. This rule applies across GLM, MLM, PCA, GRM, clustering, meta-analysis and plotting workflows.

OptionPurposeRecommendation
--outPrefix for result tables, figures and logs.Use a directory plus a descriptive prefix, for example result/trait1_glm.
--threadsNumber of CPU threads.Use a value that leaves resources for the operating system and other jobs.
--stepNumber of markers loaded into RAM at a time.Increase for speed when memory is sufficient; decrease when memory is tight.

Output prefix rule. MVP appends suffixes such as .assoc.txt or plot file extensions to the prefix. The prefix itself should not include those suffixes.

GLM Analysis

Use --glm as the first GWAS scan to verify phenotype, genotype, covariate and association-output formatting.

Requires: phenotype file and one genotype prefix. See Phenotype File and Genotype Files.

mvp --glm \
    --bfile demo \
    --pheno demo.phe \
    --pheno-pos 2 \
    --dcovar 4,5 \
    --qcovar 6 \
    --maf 0.01 \
    --out result/glm_demo
  • Output: association table under the selected --out prefix.
  • Use --add for additive genetic effect or --dom for dominant effect. If neither is supplied, additive effect is used.
  • Add covariates only after confirming their column positions and data types in the input-format section.

MLM Analysis

Use --mlm when population structure or kinship may inflate GLM statistics. If no --grm is provided, MVP can construct the relationship matrix from genotype data.

Requires: phenotype file, genotype prefix and optionally a GRM prefix. See Relationship Matrix File.

mvp --mlm \
    --bfile demo \
    --pheno demo.phe \
    --pheno-pos 2 \
    --npc 3 \
    --out result/mlm_demo
  • Output: association table and model logs under the selected --out prefix.
  • --npc adds principal components to the fixed-effect part of the GWAS model.
  • --max-iter controls the maximum number of variance component iterations.
  • For production MLM runs, keep the GRM prefix, version and command line together with the output.

Multi-trait Analysis

Pass multiple phenotype columns to analyze several traits in one command.

Requires: one phenotype file whose selected trait columns share the same individual ID column.

mvp --mlm \
    --bfile demo \
    --pheno demo.phe \
    --pheno-pos 2,3,4 \
    --out result/mlm_multi_trait
  • Output: trait-specific columns in the association result table.
  • Each selected trait may have missing values encoded as NA.
  • If --vg and --ve are supplied, provide one value per selected trait in the same order as --pheno-pos.

PCA

Use --pca with genotype data or a pre-computed GRM to inspect population structure.

Requires: genotype prefix or GRM prefix, plus the requested number of PCs.

mvp --pca --bfile demo --npc 5 --out result/pca
mvp --pca --grm demo_grm --npc 5 --out result/pca_from_grm
  • Output: PC scores and variance summary files.
  • Use PCA output for structure review, plotting and clustering input.

Model correction. For GLM or MLM, specify --npc in the association command rather than passing a PC file with --pc.

GRM Construction

Use --make-grm to construct reusable genomic relationship matrix files.

Requires: genotype prefix. The resulting GRM can be reused by MLM, PCA and clustering workflows.

mvp --make-grm \
    --bfile demo \
    --grm-method 1 \
    --out result/demo_grm
  • Output: GRM binary matrix and ID files under the selected prefix.
  • --grm-method selects the relationship-matrix construction method.
  • Use --bitwise-grm for BED-based raw additive GRM construction when that optimized path is appropriate.
  • Use --float-prec if you need lower memory use for matrix-heavy steps.

Population Clustering

Use --cluster or --auto-cluster for HDBSCAN-based clustering from genotype, GRM or PC input.

Requires: one structure input source. See Principal Component File and Relationship Matrix File.

mvp --cluster --bfile demo --npc-cluster 3 --out result/cluster
mvp --cluster --pc demo.pc --npc 3 --out result/cluster_from_pc
  • Output: cluster labels and optional parameter/embedding files.
  • Use clustering when known population labels are unavailable or need cross-checking.
  • When clustering from genotype data, MVP first derives the requested number of PCs for clustering.
  • Use --npc-cluster for the number of PCs used in clustering from genotype or GRM input.
  • --use-umap can add UMAP before clustering; tune --umap-nn and --umap-md only after the basic command works.
  • --use-kdtree can reduce memory cost for clustering neighborhood queries.
mvp --auto-cluster \
    --bfile demo \
    --npc-cluster 10 \
    --use-umap \
    --umap-nn 15 \
    --umap-md 0.1 \
    --out result/cluster_umap

Parameter compatibility. Do not combine --grid with --use-umap. Grid optimization is for the automatic clustering workflow without UMAP.

GPU Acceleration

GPU options are available only when MVP is built with CUDA support. Use GPU acceleration mainly for computationally intensive MLM workflows and benchmark CPU versus GPU results on a representative subset before running a full dataset.

Requires: CUDA-enabled MVP build and a working CUDA runtime.

mvp --mlm \
    --bfile demo \
    --pheno demo.phe \
    --device cuda \
    --gpus 0,1 \
    --evd-method mg \
    --out result/mlm_gpu
OptionValuesUse
--deviceauto, cpu, cudaSelects compute device.
--gpusComma-separated IDsSelects GPU devices, for example 0,1.
--evd-methoddc, mg, cs, bk, jacobi, svd_zControls eigendecomposition path accepted by current runtime validation.
--mlm-var-methodai, he, hiControls variance estimation strategy.
--mlm-pathmkm, emmaxSelects MLM algorithmic path.

Runtime validation. The current source tree still has a few differences between CLI help text and later parameter validation. This table lists values that pass the current runtime checks.

Benchmark Mode

Use benchmark mode to write phase-level timing information. This is useful when comparing CPU and GPU settings, tuning --step, or preparing a performance report for a release.

Requires: any valid analysis command plus benchmark output options.

mvp --mlm \
    --bfile demo \
    --pheno demo.phe \
    --benchmark \
    --benchmark-out result/mlm_benchmark \
    --out result/mlm

MVP writes <benchmark-out>.timing.json. Keep this JSON file with the association output so runtime comparisons can be reproduced later.

  • Run benchmark commands on an otherwise quiet machine when possible.
  • Use the same input files and model options when comparing two devices.
  • Record CPU model, GPU model, memory size and MVP version together with the timing file.

Plot Generation During Analysis

When the plotting module is enabled, add --plot to generate figures after analysis. This is convenient for routine GWAS runs because association tables and plots are generated from the same command and output prefix.

Requires: a plotting-enabled MVP build and an analysis that produces plot-ready output.

mvp --glm \
    --bfile demo \
    --pheno demo.phe \
    --plot \
    --plot-theme nature \
    --out result/glm_plot
  • Output: association table plus figures under the selected prefix.
  • Use --plot-only from an existing result file when you only need to regenerate figures.
Section 4

Meta-analysis

MVP can combine GWAS summary statistics from multiple cohorts or populations. The standalone mode uses --meta and --sumstat.

Overview

Meta-analysis starts from existing GWAS summary statistics. Use this workflow when each cohort or population has already been analyzed separately.

Requires: at least two summary statistics files. See GWAS Summary Statistics File.

mvp --meta \
    --sumstat cohort1.sumstat,cohort2.sumstat,cohort3.sumstat \
    --meta-method 1 \
    --maf 0.01 \
    --out result/meta_demo
  • Output: combined association results under the selected --out prefix.
  • Use comma-separated paths in --sumstat; do not insert spaces around commas.
  • Use --maf to remove low-frequency markers before combining evidence.

Summary Statistics Input

The file format is defined once in GWAS Summary Statistics File. In meta-analysis commands, pass multiple summary files as a comma-separated list. 

--sumstat cohort1.sumstat,cohort2.sumstat,cohort3.sumstat
Pre-checkReasonRecommended action
Genome buildPosition mismatches create false marker alignment.Convert all cohorts to the same build before meta-analysis.
Allele orientationEffect directions depend on a1 and a2.Harmonize effect alleles and frequencies across cohorts.
Sample sizeFixed and random effects depend on reliable standard errors and sample sizes.Use cohort-specific numeric n values.
P-value rangeInvalid p-values break p-value aggregation methods.Ensure 0 < p <= 1 whenever possible.

Pre-check. Harmonize marker IDs, genome build, allele orientation and sample-size columns before combining cohorts.

Meta Method Selection

SelectorMethodUse when
--meta-method 1UCALMSummary-statistics integration for heterogeneous populations.
--meta-method 2Fixed effectCohorts are similar and effect directions are expected to match.
--meta-method 3Random effectEffect sizes may vary across cohorts.
--meta-method 4MR-MEGAAllele-frequency structure should be modeled across populations.
--meta-method 5ACATSignals may appear in only a subset of cohorts.

UCALM shortcut. Use --ucalm only for the full individual-level UCALM workflow. For existing summary statistics, use --meta --meta-method 1.

Meta Method Recipes

Switch methods by changing only --meta-method and the output prefix.

# Fixed effect
mvp --meta --sumstat cohort1.sumstat,cohort2.sumstat --meta-method 2 --out result/meta_fe

# Random effect
mvp --meta --sumstat cohort1.sumstat,cohort2.sumstat --meta-method 3 --out result/meta_re

# MR-MEGA
mvp --meta --sumstat cohort1.sumstat,cohort2.sumstat --meta-method 4 --out result/meta_mrmega

# ACAT
mvp --meta --sumstat cohort1.sumstat,cohort2.sumstat --meta-method 5 --out result/meta_acat

Subgroup GWAS to Meta-analysis

When --pop-class is supplied with GLM or MLM, MVP can run subgroup analyses and then integrate subgroup results through the meta-analysis workflow when the meta module is enabled. 

mvp --glm \
    --bfile demo \
    --pheno demo.phe \
    --pop-class demo.pop \
    --out result/subgroup_gwas

mvp --meta \
    --sumstat result/subgroup1.sumstat,result/subgroup2.sumstat \
    --meta-method 1 \
    --out result/subgroup_meta

Use this pattern when known population labels are available. For input-file details, see Population Classification File.

  • Population labels should match the study design, such as cohort, breed, ancestry group or experimental batch.
  • Do not use --cluster and --pop-class in the same command.
  • If subgroup sample sizes are very small, inspect subgroup association results before interpreting combined meta-analysis output.

Meta-analysis Output

Meta-analysis writes association output with marker columns and method-specific p-value or effect columns under the selected output prefix.

Output componentDescriptionHow to inspect it
Marker columnsSNP, CHR, POS, A1, A2.Confirm marker alignment and allele orientation.
Method columnsCombined statistics generated by the selected meta method.Use the p-value column corresponding to the selected method.
Subgroup columnsWhen --pop-class is used, subgroup-specific statistics may be included.Compare subgroup effects with combined results.

Result review. Large blocks of missing values usually indicate marker mismatch, allele mismatch or aggressive filtering in one or more cohorts.

Section 5

UCALM

UCALM stands for Unsupervised adaptive Clustering-based Association and Layered Meta integration.

Individual-level Data Workflow

Use --ucalm when starting from genotype and phenotype files. This path covers clustering, subgroup GWAS and layered integration.

Requires: genotype prefix, phenotype file and optional covariates. See Genotype Files and Phenotype File.

mvp --ucalm \
    --bfile demo \
    --pheno demo.phe \
    --pheno-pos 2 \
    --out result/ucalm_full
  • Output: subgroup analysis results plus final UCALM integration output under the selected prefix.
  • Use this path for complex populations when predefined labels are incomplete or too coarse.

Summary-statistics Workflow

Use this path when subgroup or cohort summary statistics already exist and only the UCALM integration stage is needed.

Requires: summary statistics prepared as described in Summary Statistics Input.

mvp --meta \
    --sumstat pop1.sumstat,pop2.sumstat,pop3.sumstat \
    --meta-method 1 \
    --threads 8 \
    --out result/trait1_ucalm_meta
  • Output: UCALM integration results under the selected --out prefix.
  • Use this path only when grouping and subgroup-level GWAS have already been completed.

UCALM Output and Interpretation

UCALM writes method-specific association results under the selected --out prefix. The forward-facing output column name should be treated as UCALM_PVAL.

  • Sort output by the UCALM_PVAL column to identify top markers.
  • Output: association result table plus any subgroup-level files produced by the chosen workflow.
  • Check whether top markers have consistent chromosome, position and allele fields across cohorts.
  • Compare component evidence from FE, MR-MEGA and ACAT to classify likely effect patterns.

Interpretation. Review cohort-level effect direction, sample size and QC before claiming a biologically meaningful UCALM signal.

Practical Notes

  • Use consistent genome builds and allele coding across subgroups or cohorts.
  • Keep clustering results, subgroup sample sizes and subgroup GWAS commands together with final UCALM outputs.
  • Check MAF, sample-size filtering and rare-variant behavior before interpreting UCALM results.
  • Small subgroups can reduce the power of subgroup-specific association tests; avoid over-interpreting signals from underpowered clusters.
  • UCALM can detect statistical heterogeneity, but functional interpretation still requires annotation and biological evidence.
ProblemLikely causeFix
Many missing meta resultsMarkers do not overlap or are filtered out.Check marker IDs, genome build, MAF and sample size.

GLM, MLM and meta-analysis write association results under the --out prefix. Keep the corresponding log file together with result tables for reproducibility.

GRM Output Files

GRM construction writes binary matrix and ID files that can be reused by --mlm or --pca. 

result/demo_grm.bin
result/demo_grm.id
  • .bin stores the lower triangle of the relationship matrix in binary format.
  • .id stores the individual order used by the binary matrix.
  • Always keep .bin and .id together; moving only one file will break reuse.
  • When filtering samples, rebuild or carefully match the GRM to the filtered sample set.

PCA writes eigenvalue and eigenvector outputs. Use these files to inspect population structure or to support downstream reports.

Meta-analysis Output Files

Meta-analysis output includes marker metadata and method-specific combined statistics. Check cohort-level inputs when results contain many missing values. 

result/meta_ucalm.assoc.txt
  • Standalone meta-analysis output begins with marker metadata columns.
  • Method-specific columns follow the marker columns and depend on --meta-method.
  • When subgroup GWAS is triggered by --pop-class, output can include subgroup-specific MAF, beta, SE and p-value columns.
  • For reporting, keep the list of cohort summary files and the exact method number together with the output table.

Manhattan Plot

Use --plot after GWAS or --plot-only from an existing association file. Manhattan plots use chromosome, position and p-value columns from the association table. 

mvp --plot-only \
    --plot-input result/glm_demo.assoc.txt \
    --plot-format png \
    --plot-thresh 5e-8 \
    --plot-suggest 1e-5 \
    --out result/glm_demo_manhattan
  • Use --plot-thresh for the genome-wide significance line.
  • Use --plot-suggest for a suggestive threshold line.
  • Use --plot-filter or --no-plot-filter to control SNP downsampling for very large datasets.

Q-Q Plot

Q-Q plots include expected and observed p-value distributions and can help diagnose inflation. A strong upward deviation across most markers may indicate population structure, relatedness, batch effects or model misspecification.

  • Compare GLM and MLM Q-Q plots to evaluate whether MLM reduces inflation.
  • Inspect lambda GC when available in the plot output.
  • Do not judge model quality only by the most significant tail; inspect the overall p-value distribution.

PCA Plot

PCA plotting summarizes PC scores and variance explained when plotting is enabled for PCA output or plot-only mode.

  • Review PC1 versus PC2 first, then inspect additional PCs if the dataset has complex structure.
  • Use known population labels outside MVP if you need annotated group colors in downstream figures.
  • Large outliers should be checked against sample metadata and genotype QC.

Density and Circular Plots

MVP supports SNP density plots and circular Manhattan plots when the plotting module is enabled. Use --plot-circular to request circular Manhattan output.

  • Density plots help identify marker coverage differences across chromosomes.
  • Circular Manhattan plots are useful for compact summaries in reports or presentations.
  • Use vector formats such as pdf or svg when the figure will be edited after generation.

Plot-only Mode

mvp --plot-only \
    --plot-input result/glm.assoc.txt \
    --plot-format png \
    --out result/glm_plot

Plot-only mode skips association analysis and reads an existing association result file. It is useful when you want to test different plot sizes, themes, thresholds or formats without rerunning GWAS.

Input format. Plot-only mode expects an association table with marker, chromosome, position and p-value columns in the MVP output style.

Benchmark Output

Benchmark mode writes <benchmark-out>.timing.json with phase-level timing such as data loading, GRM construction, eigendecomposition, variance estimation and marker scanning. 

result/demo_mlm_cpu.timing.json
  • Use benchmark output to find whether time is spent in loading, GRM construction, eigendecomposition or marker scanning.
  • Use separate benchmark prefixes for CPU and GPU runs.
  • Keep hardware and software environment notes with benchmark JSON files.

Common Input Errors

  • Phenotype IDs do not match genotype IDs.
  • --bfile or --pfile is supplied with a file extension instead of a prefix.
  • Phenotype column positions are counted incorrectly.
  • Missing values are not encoded as NA.
  • Summary statistics files use inconsistent allele coding across cohorts.

Common Runtime Errors

  • Using --bfile and --pfile together is not allowed.
  • Using --keep and --remove together is not allowed.
  • Using --extract and --exclude together is not allowed.
  • --pc is intended for standalone clustering input, not ordinary GLM or MLM correction.
  • Optional flags may be unavailable if clustering, meta-analysis or plotting modules are disabled at build time.
